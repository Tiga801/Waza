# Design：带着主张去构建

生产级 UI，配以坚定的视觉方向。如果效果看起来像默认提示词生成的，就不够好。

## 先锁定方向

在写第一行代码之前，用 AskUserQuestion 获取以下四个问题的答案：

1. **谁在使用这个，在什么场景下？** 分析师查看的仪表盘与落地页或引导流程截然不同。
2. **美学方向是什么？** 精确命名：密集编辑感、原始终端风、多层玻璃拟态、纸墨感、野兽派网格、温暖模拟感。"简洁现代"不是一个方向。
3. **这个页面留在记忆里的是什么？** 一种独特的字体、一个出乎意料的色彩系统、一个令人惊喜的动效、一个非对称布局。选定一个并让它显而易见。
4. **硬性约束是什么？** 框架、包大小、对比度最低要求、键盘可访问性。

四个问题全部回答之前不得继续。在写任何代码之前，用一句话说明所选方向。

然后加载 `references/design-reference.md`，检查技术栈冲突表。在写第一个组件之前，确定唯一的 CSS 策略。

## 字体排版

字体是最响亮的设计决策。将一款展示字体与一款正文字体配对。展示字体应承载整个页面的美学重量：选择一款个性鲜明、经过主动选择而非默认使用的字体。

不要将 Inter、Roboto 或 system-ui 作为主要展示字体。它们是透明的。

## 色彩

建立一套系统，而不是一堆色值。使用 CSS 变量。选定一个主导色统领整个页面，再选一两个与之对抗的强调色。均匀分布在色板上是安全的选择；安全的选择看起来毫无设计感。

## 动效与交互

用动效创造存在感和层次感，而非噪音。对于视觉主导的作品，目标至少三处有意为之的动效：英雄区的一段入场序列、一处滚动联动或深度效果、一处强化可供性的悬停或揭示效果。如果使用 React，考虑动效库。纯 HTML 用 CSS transitions 和 `@keyframes` 已经足够。

时间：入场动画 200-400ms，退场 150-250ms。入场用 ease-out，退场用 ease-in。检验标准：如果去掉动画，页面的使用感丝毫不变，那这个动画只是装饰。去掉它。

## 布局

网格是起点，不是规则。有控制的非对称、元素叠压和斜向强调，能打造出有设计感的界面。选择一种空间策略：充裕的留白或紧凑的信息密度。不要两者折中。

背景应该有氛围：网格渐变、细微噪点、几何纹理、多层景深。纯白或纯黑几乎不是正确的选择。

默认不用卡片分区。如果去掉容器的边框、阴影和背景，理解或交互不受影响，那它就不是卡片。

如果美学方向与移动端约束冲突（例如低功耗设备上的玻璃拟态、小屏幕上的密集编辑布局），明确说出这个权衡并决定哪个优先。

## 代码质量

- CSS 变量覆盖每个设计 token：颜色、间距、字号比例、圆角
- 占位内容必须附注说明用什么替换
- 全程使用语义化 HTML
- 从一开始就做响应式：说明是移动优先还是桌面优先
- 对比度至少通过 WCAG AA 标准

实现复杂度必须与视觉野心匹配。不要为分层设计写稀薄的代码，也不要为简约设计过度工程化。

提交之前，重新加载 `references/design-reference.md` 并检查常见陷阱列表。然后请用户在浏览器中打开结果并确认看起来正确。在脑子里渲染正确但在浏览器中出问题的代码，不算完成。

## 常见陷阱

来自历史会话的真实失败，按频率排序：

- **使用了 Inter 作为展示字体。** 这是阻力最小的路径。Inter 是透明的：它对设计方向什么都没说。选一款有个性的字体。
- **生成了模板，不是设计。** 三张卡片，相同的阴影、相同的内边距、相同的圆角。换掉内容，布局纹丝不动。那是模板。重做。
- **宣称效果正确，却没打开浏览器。** 在脑子里渲染正确的代码，在浏览器里可能仍然显示出问题。打开它，确认它，再交付。
- **选了玻璃拟态，却无视移动端约束。** `backdrop-filter` 在低功耗设备上开销很大。选了方向却没有说明权衡。明确说出冲突并决定哪个优先。
- **选了不坚定的颜色。** 一个"百搭"的柔和蓝灰，结果什么都配不上。主导色加一个锐利的强调色。坚定下去。
- **状态切换时布局抖动。** 在空闲/连接中/活跃之间切换时，因为每个状态的元素尺寸不同导致 UI 位移。所有状态必须占用相同的布局空间；使用固定尺寸或 `min-height`。

## 交付

结尾包含：
- 美学方向，用两三句话命名并说明理由
- 任何非显而易见的选择解释：字体选择、色彩决策、布局逻辑
- 用真实内容替换占位内容的明确说明

交付后停止。除非用户要求修改，否则不再迭代。
