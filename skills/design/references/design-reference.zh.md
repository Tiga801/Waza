# 设计参考

## 技术栈冲突

以下组合会产生静默失败或混乱输出。永远不要组合使用：

| 不要组合 | 原因 |
|----------|------|
| Tailwind + CSS Modules 作用于同一元素 | 优先级冲突，级联不可预测 |
| Framer Motion + CSS transitions 作用于同一元素 | 对同一属性双重动画导致卡顿 |
| styled-components 或 emotion + Tailwind | 两套相互竞争的 class 系统争夺同一 DOM 节点 |
| Heroicons + Lucide + Font Awesome 混用于同一项目 | 视觉不一致、尺寸错位、包体积膨胀 |
| 多个 Google Font 字体族作为展示字体 | 相互竞争的个性互相抵消 |
| 玻璃拟态 backdrop-filter + 实线 `border: 1px solid` | 实线边框会打破多层景深的视觉幻觉 |
| 深色背景 + 全不透明 `#ffffff` 文字 | 过于刺眼；使用 `rgba(255,255,255,0.85)` 或 `#f0f0f0` |

在写第一个组件之前，确定项目唯一的 CSS 策略：仅 Tailwind、仅 CSS Modules，或仅 CSS-in-JS。不要偏离它。

## 常见陷阱

提交之前，检查以下内容是否在无意中出现：

- 白色背景上的紫色或蓝色渐变作为英雄区背景
- 三段式英雄区：大标题、单行副文本、并排两个 CTA 按钮
- 一组卡片，相同的圆角、相同的阴影、相同的内边距
- 顶部导航栏，logo 居左、链接居中、主操作靠右
- 各区块在白色和 `#f9f9f9` 之间交替
- 图标或插图居中，下方跟着标题，再下方跟着段落
- 四列页脚，各列权重相同

以上任何一种，如果是有意为之地服务于设计，都可以出现。它们不能默认出现。

最终检验：如果换入完全不同的内容，布局不作任何修改仍然说得通，那你构建的是模板，不是设计。重做。
